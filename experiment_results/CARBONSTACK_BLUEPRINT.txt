三、每個項目的詳細資訊
═══════════════════════════════════════════════════════════════

#1 carboncode — C++ 優化引擎 ★★★ 當前重點
   狀態：已有實驗結果
   Dataset：PIE (978 tasks), CarbonBench v2 (10 tasks)
   成果：78% pass@1, 1244x speedup (Decompose)
   論文目標：NeurIPS 2026
   產品化：air-gapped C++ optimization agent
   相關論文：TRACE (2026), MARCO (2025), PIE (ICLR 2024)

#2 carbon-verify — UVM Testbench 自動生成
   狀態：概念階段
   Dataset 需求：VerilogEval, OpenCores, PyUVM
   產業價值：★★★★★ (驗證工程師 60% 時間在寫 testbench)
   相關論文：AssertLLM (2024), AutoChip (2024)
   與 #1 的關係：生成的 testbench 可用 carboncode 優化

#3 carbon-hls — HLS C++ 生成
   狀態：概念階段
   Dataset 需求：Rosetta Benchmarks, Sodalite, Vivado HLS 範例
   產業價值：★★★★ (MediaTek 大量用 HLS)
   相關論文：ChipNeMo (NVIDIA, 2023)
   與 #1 的關係：HLS 輸出的 C++ 可用 carboncode 優化效能

#4 carbon-firmware — Firmware/Driver 自動生成
   狀態：概念階段
   Dataset 需求：Zephyr RTOS, Linux Kernel Drivers (I2C/SPI/UART)
   產業價值：★★★★ (每個晶片都需要 driver)
   與 #1 的關係：生成的 firmware C++ 可用 carboncode 優化

#5 carbon-sim — Simulation 加速
   狀態：概念階段
   Dataset 需求：Gem5, Verilator traces
   產業價值：★★★★★ (tape-out 前跑幾百次 sim，時間=錢)
   與 #1 的關係：本質上就是 carboncode 應用在 simulator code

#6 carbon-legacy — Legacy Code 重構
   狀態：可從 carboncode 直接延伸
   Dataset 需求：GitHub historical C/C++, MISRA compliance datasets
   產業價值：★★★ (10 年以上老 code 維護)
   與 #1 的關係：共用同一個 pipeline，只是 prompt 不同

#7 carbon-translate — Python→C++ 翻譯
   狀態：有理論基礎（TRACE paper）
   Dataset 需求：MBPP (Python) mapped to C++, TRACE benchmark
   產業價值：★★★ (Python 原型→C++ 產品化)
   與 #1 的關係：翻譯後的 C++ 可用 carboncode 優化
   學術角度：解決「多對一映射」問題（concept.md 的核心洞察）

#8 carbon-debug — 自動 Debug & Root Cause
   狀態：概念階段
   Dataset 需求：Bugs2Fix, GitHub Issue/Commit pairs
   產業價值：★★★★ (debug 佔設計時間 40%)
   與 #1 的關係：debug 結果可用 carboncode 的 fix loop

五、共用的核心技術
═══════════════════════════════════════════════════════════════

所有 8 個項目共用的 shared/ 元件：

1. LLM Interface (shared/llm/)
   - Ollama wrapper, prompt templates
   - 支援 Qwen2.5-Coder-7B, 未來換更大模型

2. Compiler Pipeline (shared/compiler/)
   - compile + ASan + UBSan + benchmark
   - 確定性驗證（mechanical coupling）

3. Preflight Checks (shared/preflight/)
   - bits/stdc++.h 處理
   - IO 優化
   - 巨集展開
   - Missing include 自動補

4. Two-Pass Architecture (shared/pipeline/)
   - Pass 1: 保持正確性
   - Pass 2: 優化
   - Fallback: 多路徑 preprocessing
   - Speedup guard: 不比原始慢

5. Verification Framework (shared/verify/)
   - Multi-test-case validation
   - Output semantic matching
   - Checkpoint/resume

═══════════════════════════════════════════════════════════════

「CarbonStack：用本地 7B 模型自動化半導體 C++ 開發全流程」

七、Gemini 的建議
═══════════════════════════════════════════════════════════════

1. 用 Git Submodules 管理（每個 carbon-* 可獨立發布）
2. CarbonCode 是 North Star — 所有其他項目產出的 code 都經過它優化
3. 不要同時做 8 個 — 先把 #1 做到 NeurIPS 水準
4. #6 (Legacy) 和 #7 (Translate) 最容易從 #1 延伸
5. #2 (UVM) 是產業價值最高的，但需要 dataset
